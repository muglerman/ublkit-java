Flujo tipico con UBLKit

  1. 1. Modelar

    Construye el borrador de documento desde el dominio de negocio.

  2. 2. Ensamblar

    Auto-calcula impuestos, totales y reglas documentales.

  3. 3. Validar

    Aplica validacion funcional y opcionalmente XSL de referencia SUNAT.

  4. 4. Firmar

    Firma XML con certificado X509 para envio formal.

  5. 5. Enviar y Renderizar

    Publica hacia SUNAT/OSE y genera HTML, PDF o ticket.

Arquitectura modular

Cada modulo cumple una responsabilidad puntual para evitar acoplamiento y simplificar evolucion.

ublkit-core

Tipos, value objects, resultados y errores base.

ublkit-catalogs

Catalogos normativos SUNAT y consultas por codigo.

ublkit-validation

Validadores funcionales y homologacion XSL.

ublkit-ubl

Modelado documental y serializacion XML UBL 2.1.

ublkit-sign

Firma digital XML con RSA y certificados PKCS12.

ublkit-gateway

Envio y consulta SOAP/REST para SUNAT y GRE.

ublkit-render

Plantillas HTML, PDF y tickets 58mm/80mm.

ublkit-qr

Generacion de QR SUNAT para representacion impresa.

ublkit-storage

Persistencia de XML, CDR y documentos derivados.

ublkit-testkit

Fixtures, simuladores y aserciones para pruebas.

ublkit-spring-boot-starter

Autoconfiguracion para proyectos Spring Boot.

ublkit-quarkus

Productores CDI y arranque rapido con Quarkus.

Arranque rapido

Si trabajas con Spring Boot, puedes iniciar con el starter oficial y escalar luego por modulo.

  • Define emisor y receptor desde el dominio.
  • Ensamblar para impuestos y totales.
  • Validar, serializar, firmar y enviar.
<dependency>
  <groupId>com.cna</groupId>
  <artifactId>ublkit-spring-boot-starter</artifactId>
  <version>0.1.0-SNAPSHOT</version>
</dependency>
