Libreria comunitaria para UBL 2.1 en Java

Modela, valida, firma y envia documentos electronicos con una arquitectura modular.

UBLKit te ayuda a construir un flujo completo para Factura, Boleta, Notas, Guia, Resumen, Baja, Percepcion y Retencion, con enfoque de dominio y homologacion SUNAT.

Flujo tipico con UBLKit

  1. 1. Modelar

    Construye el borrador de documento desde el dominio de negocio.

  2. 2. Ensamblar

    Auto-calcula impuestos, totales y reglas documentales.

  3. 3. Validar

    Aplica validacion funcional y opcionalmente XSL de referencia SUNAT.

  4. 4. Firmar

    Firma XML con certificado X509 para envio formal.

  5. 5. Enviar y Renderizar

    Publica hacia SUNAT/OSE y genera HTML, PDF o ticket.

Modulos desglosados

Cada modulo cumple una responsabilidad puntual. Explora en detalle que trae cada uno y como integrarlo.

ublkit-core

Base transversal de todo UBLKit. Define tipos de dato, enums, value objects y excepciones del framework.

Beneficios

  • Tipado fuerte sin dependencias externas
  • Enums para tipos de documento, ambiente, protocolo
  • Value Objects para Dinero, Moneda, NumeroSerie
  • Resultado generico para operaciones consistentes

Casos de uso

Define el contrato entre capas. Si creas un adaptador personalizado (p.ej. para otra OSE), extiendes desde aqui.

// Value Object tipado
Dinero impuesto = new Dinero(
  new BigDecimal("18.00"),
  Moneda.PEN
);

// Enum de tipos
TipoDocumento tipo = TipoDocumento.FACTURA;
TipoAmbiente ambiente = TipoAmbiente.BETA;

ublkit-catalogs

Gestiona los 60+ catalogos normativos SUNAT (codigos de pais, impuestos, monedas, etc.) con busqueda rapida y cache.

Beneficios

  • Busqueda O(1) por codigo en 35+ catalogos CSV embebidos
  • Cache concurrente para rendimiento en tiempo real
  • Validacion de existencia de codigos SUNAT
  • Soporte multidireccional (codigo → descripcion y viceversa)

Casos de uso

Antes de emitir, valida que el codigo de tributacion, moneda o departamento exista en los catalogos oficiales.

ProveedorCatalogos catalogos = new ProveedorCatalogos();

// Validar codigo de impuesto
String descImpuesto = catalogos
  .buscarPorCatalogo("05", "1000");
// Retorna: "IGV - Impuesto General a las Ventas"

// Validar pais
boolean esPeruValido = catalogos
  .existeEnCatalogo("03", "PE");

ublkit-ubl

Mapeo integral del modelo de dominio al standard XML UBL 2.1. Incluye serializadores y ensambladores con auto-calculo de impuestos.

Beneficios

  • Modelos de dominio amigables (BorradorFactura, BorradorNota)
  • Auto-calculo de IGV, ISC, ICBPER y totales
  • Ensambladores que aplican reglas de negocio SUNAT
  • Serializacion XML nativa (javax.xml DOM) sin JAXB

Casos de uso

Punto central para transformar datos de negocio a XML oficial. Si necesitas customizar el template XML, aqui es el lugar.

// Crear borrador
BorradorFactura factura = new BorradorFactura();
factura.setSerie("F001");
factura.setNumero(1);

LineaDetalle linea = new LineaDetalle();
linea.setDescripcion("Servicio");
linea.setCantidad(new BigDecimal("1"));
linea.setPrecio(new BigDecimal("100.00"));

factura.setDetalles(List.of(linea));

// Ensamblar (auto-calcula IGV, totales, etc)
EnsambladorFactura.ensamblar(factura);

// Serializar a XML UBL 2.1
String xmlUbl = new SerializadorXmlFactura()
  .serializar(factura);

ublkit-validation

Validacion integral funcional + homologacion XSL oficial SUNAT. No rompe en el primer error; acumula incidencias.

Beneficios

  • 8 validadores (Factura, Boleta, Notas, Guia, Resumen, Baja, Percepcion, Retencion)
  • Validacion funcional local sin latencia SUNAT
  • Opcionalmente ejecuta XSL embebido de referencia SUNAT
  • Reporta todos los errores y advertencias, no solo el primero

Casos de uso

Valida antes de firmar y enviar. Usa homologacion XSL para garantizar cumplimiento SUNAT.

ValidadorFactura validador = new ValidadorFactura();
ResultadoValidacion resultado = validador.validar(factura);

if (!resultado.esValido()) {
  for (IncidenciaValidacion inc : resultado.getIncidencias()) {
    System.out.println(
      inc.getRegla() + ": " + inc.getMensaje()
    );
  }
}

ublkit-sign

Firma digital de XML con certificados X509 (PKCS12). Implementa RSA-SHA1 enveloped signature.

Beneficios

  • Firma enveloped (XML firmado incluye la firma)
  • Soporte PKCS12 (.pfx, .p12) nativo
  • Validacion y sanitizacion segura del DOM
  • Compatible con certificados SUNAT

Casos de uso

Obligatorio antes de enviar a SUNAT. Requiere certificado digital valido (RUC + contraseña).

// Cargar certificado
DetallesCertificado cert = CargadorCertificado.cargar(
  new OrigenCertificado(
    inputStream,
    "contraseña_cert",
    "PKCS12"
  )
);

// Firmar XML
String xmlSinFirma = "...XML UBL...";
ResultadoFirma firma = ServicioFirma.firmarXml(
  xmlSinFirma,
  "#UBLKIT-SIGN",
  cert
);

String xmlFirmado = firma.xmlFirmadoStr();

ublkit-gateway

Envio y consulta a SUNAT/OSE via SOAP y REST. Manejo de tokens OAuth2, retry con backoff y caché.

Beneficios

  • Protocolo SOAP para factura/notas/guia clasica
  • REST para GRE (Guias de Remision Electronica)
  • OAuth2 automático con refresh de token
  • Retry exponencial y manejo de timeout

Casos de uso

Envia el XML firmado a SUNAT y consulta el CDR (Comprobante de Recepcion) para confirmar aceptacion.

PasarelaSunat pasarela = new PasarelaSunatDefecto(
  credenciales,  // usuario + password SUNAT
  TipoAmbiente.BETA
);

ResultadoEnvio envio = pasarela.enviarComprobante(
  xmlFirmado,
  TipoDocumento.FACTURA
);

if (envio.esExitoso()) {
  String ticket = envio.getNumeroTicket();
  
  // Consultar CDR
  ResultadoConsulta consulta = pasarela
    .consultarTicket(ticket);
}

ublkit-render

Genera representaciones visuales: HTML, PDF y tickets termicos (58mm/80mm). Plantillas Pebble + OpenHTMLtoPDF.

Beneficios

  • Plantillas HTML (Pebble) reutilizables y personalizables
  • PDF de alta calidad con OpenHTMLtoPDF
  • Soporte ticket termal (58mm y 80mm)
  • Contexto de render unificado para todos los tipos

Casos de uso

Genera documentos de usuario final: PDF para email, HTML para web, ticket para impresora termal de punto venta.

ContextoRender contexto = new ContextoRender()
  .setFactura(factura)
  .setQrBase64(qrCodeB64)
  .setHashCdr(hashCdr);

// Renderizar a HTML
RenderizadorFacturaHtml htmlRender = 
  new RenderizadorFacturaHtml();
String html = htmlRender.renderizar(contexto);

// Renderizar a PDF
RenderizadorFacturaPdf pdfRender = 
  new RenderizadorFacturaPdf();
byte[] pdf = pdfRender.renderizar(contexto);

ublkit-qr

Genera codigos QR segun standard SUNAT. Vincula clave de acceso, montos, hash CDR y datos fiscales.

Beneficios

  • QR SUNAT oficial: clave acceso + montos + hash
  • Generacion base64 lista para embeber en PDF/HTML
  • Validacion de componentes antes de generar
  • Soporta todos los tipos de documento

Casos de uso

Embebe el QR en el PDF/ticket para que clientes validen el comprobante escaneando en portal SUNAT.

GeneradorQrSunat generador = new GeneradorQrSunat();

String qrBase64 = generador.generarQr(
  factura.getClaveAcceso(),
  factura.getSumaIgv(),
  factura.getTotal(),
  factura.getFechaEmision(),
  hashCdr
);

// Usar en contexto render
contexto.setQrBase64(qrBase64);
// <img src="data:image/png;base64,...">

ublkit-storage

Persiste artefactos: XML, CDR, PDF, tickets. Adaptadores para S3, GCS y almacenamiento local.

Beneficios

  • Interfaz unificada AlmacenDocumentos
  • Adaptadores S3, Google Cloud Storage, Local FS
  • Metadatos y busqueda por fecha, tipo, RUC
  • Versionado y historial de cambios

Casos de uso

Archiva XML firmado, CDR del SFS, PDF y ticket para auditoría y consultas posteriores.

AlmacenDocumentos almacen = new AlmacenS3(
  bucketNombre,
  credencialesAws
);

// Guardar XML
almacen.guardarXml(
  "F001-00000001",
  xmlFirmado,
  metadatos
);

// Recuperar PDF
byte[] pdf = almacen.descargarPdf(
  "F001-00000001"
);

ublkit-testkit

Herramientas para testing: fixtures de documentos, simulador de gateway, aserciones XML y CDR mocks.

Beneficios

  • Fixtures pre-construidas (FacturasEjemplo, NotasEjemplo, etc.)
  • SimuladorGateway para pruebas sin conectar SUNAT
  • Aserciones tipadas (AssertsCdr, AssertXml)
  • Respuestas SUNAT simuladas para todos los escenarios

Casos de uso

Pruebas unitarias y de integracion sin conectar SUNAT real. Tests reproducibles y rapidos.

// Usar fixture
BorradorFactura factura = FacturasEjemplo
  .facturaEstandarValida();

// Simular respuesta SUNAT
SimuladorGateway simulador = new SimuladorGateway()
  .conRespuestaExitosa();

PasarelaSunat pasarela = simulador.crearPasarela();
ResultadoEnvio resultado = pasarela
  .enviarComprobante(xml);

// Aserciones
AssertsCdr.queCdrEsValido(resultado.getCdr());
AssertXml.queExisteElemento(xml, "/Invoice/ID");

ublkit-spring-boot-starter

Integracion oficial para Spring Boot. Auto-configura beans, propiedades y esterotipos personalizados.

Beneficios

  • Autoconfiguracion zero-setup (@SpringBootApplication)
  • Propiedades YAML centralizadas (ublkit.gateway.*)
  • Inyeccion de serializadores, validadores y renderizadores
  • Health checks para conexion SUNAT

Casos de uso

Proyectos Spring Boot que necesitan emitir documentos electronicos. Es la forma mas simple de integrar.

// application.yml
ublkit:
  gateway:
    ambiente: BETA
    credenciales:
      usuario: ${SUNAT_USER}
      password: ${SUNAT_PASS}

// En un @Controller o @Service
@Autowired
private SerializadorXmlFactura serializador;

@Autowired
private PasarelaSunat pasarela;

public void emitirFactura() {
  String xml = serializador.serializar(factura);
  pasarela.enviarComprobante(xml);
}

Flujos avanzados de integracion

Ejemplos de como combinar modulos para casos reales.

🏪 Punto de Venta (POS)

Emitir facturas/boletas en tiempo real con venta y generar ticket.

ublkit-core ublkit-ubl ublkit-validation ublkit-sign ublkit-gateway ublkit-render (ticket) ublkit-qr 
// 1. Crear boleta
BorradorBoleta boleta = new BorradorBoleta();
boleta.setDetalles(carrito.getItems());

// 2. Ensamblar + Validar
EnsambladorBoleta.ensamblar(boleta);
if (!validador.validar(boleta).esValido()) return;

// 3. Serializar + Firmar
String xml = serializador.serializar(boleta);
String xmlFirmado = servicioFirma.firmarXml(xml, cert);

// 4. Enviar async
completableFuture(() -> 
  pasarela.enviarComprobante(xmlFirmado)
);

// 5. Renderizar ticket local
byte[] ticket = renderizadorTicket.renderizar(boleta);
impresora.imprimir(ticket);

📦 E-commerce

Emitir factura post-compra, generar PDF y archivar en S3.

ublkit-core ublkit-catalogs ublkit-ubl ublkit-validation ublkit-sign ublkit-gateway ublkit-render (PDF) ublkit-storage (S3) 
// 1. Crear factura desde pedido
BorradorFactura factura = new BorradorFactura();
factura.setReceptor(pedido.getCliente());
factura.setDetalles(pedido.getLineas());

// 2. Ensamblar + Validar
EnsambladorFactura.ensamblar(factura);
validador.validar(factura);

// 3. Firma + Envio
String xmlFirmado = servicioFirma.firmarXml(
  serializador.serializar(factura), cert);
pasarela.enviarComprobante(xmlFirmado);

// 4. Renderizar PDF + Archivar
byte[] pdf = renderizadorPdf.renderizar(factura);
almacenS3.guardarPdf(factura.getId(), pdf);

// 5. Email a cliente
enviador.enviarPdfAlCliente(
  pedido.getCliente().getEmail(), pdf);

📊 Sistema Nomina

Generar reportes de Percepcion/Retencion con validacion SUNAT.

ublkit-core ublkit-catalogs ublkit-ubl ublkit-validation ublkit-gateway ublkit-render (HTML) 
// 1. Generar Percepcion
Percepcion percepcion = new Percepcion();
percepcion.setPeriodo("2026-04");
percepcion.setTrabajadores(empleados);

// 2. Validar reglas SUNAT
ValidadorPercepcion validador = new ValidadorPercepcion();

if (resultado.esValido()) {
  // 3. Render HTML para envio

}

Preguntas frecuentes

Respuestas a dudas comunes de implementacion.

¿Puedo usar UBLKit sin Spring Boot? +

Si, absolutamente. UBLKit es agnóstico del framework. Los módulos core, ubl, validation, sign, gateway y render funcionan con cualquier stack Java. Spring Boot starter es solo para conveniencia en proyectos Spring.

¿Cual es el overhead de dependencias? +

Minimo. ublkit-core no tiene dependencias. Los demás módulos usan librería estándar (javax.xml, java.net.http, Jackson para JSON). Opcionalemente OpenHTMLtoPDF para PDF. Sin JAXB, sin frameworks pesados.

¿Puede validar homologación SUNAT oficial? +

Si. El módulo ublkit-validation incluye XSL embebido (ValidaExprRegFactura, ValidaExprRegBoleta, etc) de referencia SUNAT. Actívalo con -Dublkit.validation.sunat.enabled=true en propiedades.

¿Soporta compilación GraalVM nativa? +

Parcialmente. Los módulos core, ubl, catalogs y validation funcionan en GraalVM. El módulo gateway requiere runtime reflection para SOAP, por eso Quarkus usa productores CDI para inyectar sin reflexión.

¿Como manejo reintentos y timeouts? +

ublkit-gateway ya implementa retry exponencial y caché de token OAuth2. Puedes customizar timeouts en propiedades YAML (ublkit.gateway.timeout-ms, ublkit.gateway.max-retries).

¿Puedo almacenar en GCS o Azure Blob? +

Si. ublkit-storage define interfaz AlmacenDocumentos. Ya hay adaptadores para S3 y GCS. Para Azure, extiendes la interfaz con tu implementacion (5-10 lineas de código).

¿Cual es el cost/rendimiento en producción? +

Bajo. Serialización XML: ~5ms por documento. Validación: ~2ms. Firma: ~50ms (depende CPU). Envio SUNAT: ~200-500ms (latencia red). Total estimado: 300-600ms por flujo completo en hardware moderno. RAM: ~50MB startup.

¿Como hago testing sin conectar SUNAT real? +

Usa ublkit-testkit. Proporciona SimuladorGateway que devuelve respuestas SUNAT simuladas sin conexión real. Perfecto para CI/CD y pruebas unitarias rápidas. Incluye fixtures de documentos válidos.

¿Hay soporte para Resumen Diario (RC) y Comunicación de Baja (RA)? +

Si. Modelos completos para ResumenDiario y ComunicacionBaja. Validadores, serializadores y ensambladores listos. Incluyen reglas SUNAT para acumulaciones y reversiones.
